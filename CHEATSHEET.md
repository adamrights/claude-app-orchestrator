# Cheatsheet

One-page reference for the common operations. Slash commands assume you've run `./install.sh` once.

## Setup

| Action | Command |
|--------|---------|
| Install slash commands | `./install.sh` |
| Uninstall | `./install.sh --uninstall` |

## Daily commands (in Claude Code)

| Action | Command |
|--------|---------|
| Build an app | `/orchestrate <blueprint.yaml> [out-dir]` |
| Validate a blueprint | `/validate <blueprint.yaml>` |
| Audit perf of an app | `/audit` (run in app dir) |
| Add a feature to an app | `/extend "<description>"` (run in app dir) |

## Daily commands (manual / shell)

| Action | Command |
|--------|---------|
| Validate one blueprint | `node scripts/validate-blueprint.mjs <blueprint.yaml>` |
| Validate every example | `make check` |
| Print the build incantation | `make build BLUEPRINT=<path> OUT=<dir>` |
| Build manually (in Claude Code) | `Read agents/orchestrator.md and build <blueprint> into <out-dir>` |

## Common errors

| Error | Fix |
|-------|-----|
| `claude-app-orchestrator not installed` | Run `./install.sh` from the repo root |
| `Unknown skill '<name>'` | Check `agents/orchestrator.md` skill mapping table; use the suggested name from the validator |
| `Cycle in feature dependencies: a → b → a` | Inspect `depends_on:` chain — two features depend on each other |
| `Template not found: <name>` | Check `templates/` — `stack.type` must map to an existing template |
| Slash command not appearing | Run `./install.sh` again; check `~/.claude/commands/` contains the symlinks |

## File layout

| Where | What |
|-------|------|
| `agents/` | Agent definitions (orchestrator + specialists) |
| `skills/` | Coding guidelines |
| `blueprints/` | Schema, examples, snippets |
| `templates/` | Starter project scaffolds |
| `examples/built/` | Reference apps generated by the orchestrator (start with `helpdesk/`) |
| `.claude/commands/` | Slash command source (symlinked into `~/.claude/commands/` by `install.sh`) |
| `scripts/` | Validator + helpers |
